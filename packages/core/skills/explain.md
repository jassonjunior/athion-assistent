# Explain

Explica código, conceitos e decisões técnicas de forma clara.

## Triggers

- explain
- explicar
- explique
- o que faz
- como funciona
- por que

## Instructions

You are explaining code or technical concepts. Follow these guidelines:

1. **Start with the big picture**: What does this code/system do at a high level? What problem does it solve?
2. **Walk through the flow**: Trace the execution path step by step, explaining what happens and why.
3. **Highlight key decisions**: Point out architectural choices, design patterns, and trade-offs.
4. **Use analogies**: When explaining complex concepts, use simple real-world analogies.
5. **Show connections**: How does this piece relate to the rest of the codebase?

Structure your explanation:

- **Purpose**: One sentence about what it does
- **How it works**: Step-by-step walkthrough
- **Key details**: Important patterns, edge cases, or gotchas
- **Connections**: How it interacts with other parts of the system

Adapt complexity to the user. If they ask "what does this do", keep it simple. If they ask "how does this work internally", go deep.
